# Versioning
This project uses a modified version of [Semantic Versioning](https://semver.org/) for versioning.

## Versioning Scheme

Each version of DEnv follows the format `DEVELOPMENT-SEQUENCE_MAJOR.MINOR.PATCH`.

### Development Sequence

The `DEVELOPMENT-SEQUENCE` is a release identifier that changes based on the software's stability, state and testing progress

#### Alpha (a)

The `a` suffix indicates that the release is still in development and may contain bugs or incomplete features.

#### Beta (b)

The `b` suffix indicates that the release is a beta version and may still contain bugs or incomplete features.

#### Nightly (n)

The `n` suffix indicates that the release is a nightly build and may contain bugs or incomplete features.

#### Release Candidate (rc)

The `rc` suffix indicates that the release is a release candidate and may still contain bugs or incomplete features.

#### Stable (s)

The `s` suffix indicates that the release is stable and has been tested for a sufficient period.

### Major, Minor, and Patch

The `MAJOR`, `MINOR`, and `PATCH` components follow the standard [Semantic Versioning](https://semver.org/) format.

#### Major (MAJOR)

The `MAJOR` component increments when there are significant changes that may break compatibility with previous versions.

#### Minor (MINOR)

The `MINOR` component increments when there are minor changes that do not break compatibility with previous versions.

#### Patch (PATCH)

The `PATCH` component increments when there are bug fixes or small improvements that do not break compatibility with previous versions.
